0-1线性数据结构

2019-11-10
Data Structures With Go
1646 words 4 min read | 阅读

Table of Contents

    • Array
    • Linked List
    • Stack
    • Queue

数据结构是在计算机中组织数据以便有效使用的一种特殊方式。这个想法是为了减少不同任务的时间和空间复杂度。以下是一些流行的线性数据结构的概述。

Array

数组是一种数据结构，用于在连续位置存储同类元素。在存储数据之前必须提供数组的大小。

令数组的大小为n：

访问时间：O(1) [这是可能的，因为元素放在连续的位置]

搜索时间：

​ 连续搜索O(n);

​ 二分查找(如果数组已排序)O(logn)

插入时间：O(n) [最坏的情况发生在，插入数组的开头，需要移动所有元素]

删除时间：O(n) [最坏的情况发生在，插入数组的开头，需要移动所有元素]

Example:

​ 假设我们要存储班级中所有学生的成绩，我们可以使用数组来存储他们。这有助于减少变量数量的使用，因为我们不需要为每个科目的分数创建单独的变量。只需遍历数组即可访问所有成绩。

Linked List

链表是线性数据结构（如数组），其中每个元素都是一个单独的对象。列表中的每个元素（即节点）都包含两项,数据和对下一个节点的引用。

链表的类型：

Singly Linked List:

​ 在这种类型的链表中，每个节点都会存储下一个节点的地址或引用，而最后一个节点的下一个地址或引用为nil。例如1-> 2-> 3-> 4-> nil。

Double Linked List:

​ 在这种链表中，每个节点有两个引用，一个引用指向下一个节点，另一个指向上一个节点。这种数据结构的优点是我们可以在两个方向上遍历，对于删除，我们不需要显式访问上一个节点。例如。nil <-1 <-> 2 <-> 3-> nil。

Circular Linked List：

​ 循环链表是一个其中所有节点都连接在一起形成一个圆的链表。最后没有nil。循环链表可以是单循环链表或双循环链表。这种数据结构的优势在于，任何节点都可以作为起始节点。这在实现链表中的循环队列时很有用。例如。1-> 2-> 3-> 1 [最后一个节点的下一个指针指向第一个节点的地址]。

元素的访问时间：O(n)

元素的搜索时间：O(n)

插入元素：O(1)

删除元素：O(1) [如果我们知道删除节点之前节点的位置]

Example:

​ 考虑前面的示例，其中我们对学生进行了一系列评分。现在，如果在课程中添加了新学科，则其分数也将添加到分数数组中。但是数组的大小是固定的，并且已经满了，因此无法添加任何新元素。如果我们将数组的大小远远大于主题数，则大多数数组可能保持空白。我们减少了空间浪费，链表仅在引入新元素时才添加节点。使用链表，插入和删除也变得更加容易。 链表的一大缺点是不允许随机访问。

使用数组，我们可以在O(1)时间访问第i个元素。在链表中，它花费O(i)时间。

Stack

Stack是一种（后进先出）的抽象数据类型，它用作元素的集合，具有两个主要操作：push（向集合中添加元素）和pop（从中删除最后添加的元素） 。在堆栈中，推入和弹出操作都发生在堆栈顶部的同一端。可以同时使用数组和链接列表来实现。

插入：O(1)

删除：O(1)

访问时间：O(n) [最坏的情况]

插入和删除都发生在集合的最后

Example:

​ 堆栈用于维护函数调用（最后一个调用的函数必须先完成执行），我们总是可以借助堆栈来删除递归。在必须反转单词，检查括号是否平衡的情况下，也会使用堆栈。同样，在Web浏览器中实现后退功能。

Queue

队列是一种先进先出的抽象数据类型，用作元素的集合。具有两个主要操作：入队（enqueue)，将元素添加到集合的过程（元素从最后添加）；出队（dequeue），即删除添加的第一个元素的过程。（该元素从最前面移除）。可以同时使用数组和链接列表来实现。

插入：O(1)

删除：O(1)

访问时间：O(n) [最坏的情况]

Example :

​ 顾名思义，队列是根据公交车站或火车的队列构建的数据结构，其中站在队列前面（站立时间最长）的人是第一个获得票证的人。因此，在多个用户之间共享资源并在先到先得的基础上提供服务的任何情况。示例包括CPU调度，磁盘调度。队列的另一种应用是在两个进程之间异步传输数据（不一定以与发送数据相同的速率接收数据）。示例包括IO缓冲区，管道，文件IO等。

Circular Queue :

​ 此数据结构的优点是，在数组实现的情况下，它减少了空间浪费，因为如果第（n + 1）个元素为空，则在第0个索引处进行插入。
