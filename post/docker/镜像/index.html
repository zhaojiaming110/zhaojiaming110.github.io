镜像

2019-10-14
Docker
1526 words 4 min read | 阅读

Table of Contents

镜像是Docker的三大核心概念之一。

Docker运行容器前需要本地存在对应的镜像，如果镜像不存在本地，Docker会尝试先从默认镜像仓库下载（默认使用Docker Hub公共注册服务器中的仓库），用户也可以配置使用自定义的镜像仓库。

获取镜像

可以使用docker pull命令从网络上下载镜像。该命令的格式为docker pull NAME[:TAG]。

对于docker镜像来说，如果不显示地指定TAG，则默认会选择latest标签，即下载最新版本的镜像。

演示：

1
2
3
4
5
6
7
8
9

$ sudo docker pull nsqio/nsq
Using default tag: latest
latest: Pulling from nsqio/nsq
5d20c808ce19: Pull complete
e43ee7addbdb: Pull complete
cbc99497dda7: Pull complete
Digest: sha256:78b986254986c4ae1237b32219a83c5a23354a6c30c18817597f776a4edcac41
Status: Downloaded newer image for nsqio/nsq:latest
docker.io/nsqio/nsq:latest

该命令实际上下载的就是nsqio/nsq:latest镜像。

下载过程可以看出，镜像文件一般由若干层组成。行首的5d20c808ce19这样的字符串代表了各层的ID。下载过程中会获取并输出镜像的各层信息。

层（Layer）其实是AUFS（一种联合文件系统）中的重要概念，是实现增量保存与更新的基础。

用户也可以通过指定标签来下载特定版本的某一个镜像：

1

$ sudo docker pull nsqio/nsq:1.3

用户也可以选择从其他注册服务器的仓库下载镜像

1

$ sudo docker pull d1.dockerpool.com:5000/nsqio/nsq

查看镜像信息

使用docker images或docker image ls命令可以列出本地主机上已有的镜像。

1
2
3
4
5
6
7

$ sudo docker images
REPOSITORY            TAG                 IMAGE ID            CREATED             SIZE
registry              2                   708bc6af7e5e        7 weeks ago         25.8MB
beuself/hello-world   latest              7cbbf0491ed0        4 months ago        2MB
mysql                 latest              c8ee894bd2bd        4 months ago        456MB
redis                 latest              de25a81a5a0b        4 months ago        98.2MB
nsqio/nsq             latest              4cac9773ca5e        6 months ago        65.6MB
  • REPOSITORY：来自于哪一个仓库，比如redis仓库。
  • TAG：镜像的标签信息，比如latest。
  • IMAGE ID：镜像的ID号（唯一）。
  • CREATED：镜像创建时间。
  • SIZE：镜像大小。

为了方便在之后使用这个镜像，可以使用docker tag命令为本地镜像添加一个新的标签。

1

sudo docker tag nsqio/nsq:latest nsq:latest

再次使用docker images列出本地主机上的镜像信息，可以看出多出了一个nsq:latest标签的镜像。

1
2
3

REPOSITORY            TAG                 IMAGE ID            CREATED             SIZE
nsq                   latest              4cac9773ca5e        6 months ago        65.6MB
nsqio/nsq             latest              4cac9773ca5e        6 months ago        65.6MB

我们可以注意到，不同标签的镜像的ID是完全一致的，说明它们实际指向了同一个镜像文件，只是别名不同而已。标签在这里起到了引用和快捷方式的作用。

docker inspect返回的是一个JSON格式的信息，如果我们只要其中一项内容时，可以使用-f参数来指定，例如，获取镜像的Os信息：

1
2

docker inspect -f {{".Os"}} 5e35e350aded
linux

在指定镜像ID的时候通常使用该ID的前若干个字符组成的可区分字串来替代完整的ID。
