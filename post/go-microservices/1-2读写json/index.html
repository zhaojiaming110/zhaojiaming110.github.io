1-2.读写json

2019-10-10

感谢encoding/json包，它内置在标准库中，可轻松快速地对go类型的JSON进行编码和解码。它实现了简单的marshal和unmarshal函数；无论如何，如果我们需要它们，包还提供了编码器和解码器类型，允许我们在读写json数据流时进行更大的控制。在本节中，我们将研究这两种方法，但首先让我们看看将标准go结构转换为相应的json字符串是多么简单。

将go结构编组为json

为了对json数据进行编码，encoding/json包提供了marshall函数，该函数具有以下签名：

1

func Marshal(v interface{}) ([]byte, error)

这个函数接受一个参数，它是interface类型的，所以几乎可以想到任何对象，因为interface表示go中的任何类型。它返回一个元组（[]byte，error），您将在go中经常看到这种返回样式，一些语言实现了try-catch方法，该方法鼓励在无法执行操作时抛出错误，go建议使用模式(return type, error)，当操作成功时，错误为零。

在go中，未处理的错误是一件坏事，虽然该语言确实实现了panic和recover，这类似于其他语言中的异常处理，但是您应该使用它们的情况却截然不同（请参阅go编程语言kernaghan）。在go中，panic函数会导致正常执行停止，go例程中所有延迟的函数调用都会执行，然后程序会因一条日志消息而崩溃。它通常用于指示代码中存在错误的意外错误，而良好的健壮go代码将尝试处理这些运行时异常并将详细的错误对象返回给调用函数。

这个模式正是用marshal函数实现的。在marshal无法从给定对象创建json编码的字节数组的实例中，这可能是由于运行时死机造成的，然后捕获该实例并将详细说明问题的错误对象返回给调用方。

让我们尝试一下，在现有的示例中展开，而不是简单地从处理程序中打印字符串，让我们为响应创建一个简单的结构，然后返回此结构。

}

func main() {
	port := 8080

	http.HandleFunc("/helloworld", helloWorldHandler)

	log.Printf("Server starting on port %v\n", port)
	log.Fatal(http.ListenAndServe(fmt.Sprintf(":%v", port), nil))
}

func helloWorldHandler(w http.ResponseWriter, r *http.Request) {
	response := helloWorldResponse{Message: "Hello World"}
	data, err := json.Marshal(response)
	if err != nil {
		panic("Ooops")
	}
    
	fmt.Fprint(w, string(data))
}

这太棒了；然而，marshall的默认行为是取字段的字面名称，并将其用作json输出中的字段。如果我更喜欢使用camel case(骆驼风格)并且更愿意看到“message”，我们可以在helloworldresponse结构中重命名该字段吗？

不幸的是，我们不能像go那样导出小写属性，marshal将忽略这些属性，并且不会将它们包含在输出中。

由于encoding/json包实现了struct field属性，允许我们将字段属性的输出更改为我们选择的任何内容。

使用struct字段的标记，我们可以更好地控制输出的外观。在前面的示例中，当我们marshal此结构时，来自服务器的输出将是：

1

{"message":"HelloWorld"}

这正是我们想要的，但是我们可以使用字段标记来进一步控制输出。如果需要，我们可以转换对象类型，甚至完全忽略一个字段：

type helloWorldResponse struct {
	// change the output field to be "message"
	Message string `json:"message"`
	// do not output this field
	Author string `json:"-"`
	// do not output the field if the value is empty
	Date string `json:",omitempty"`
	// convert output to a string and rename "id"
	Id int `json:"id, string"`
}

Channel、复杂类型和函数不能用json编码；尝试对这些类型进行编码将导致由函数Marshal返回的UnsupportedTypeError。

它也不能表示循环数据结构；如果stuct包含循环引用，那么Marshal处理将导致无限递归，这对web请求来说永远都不是好事。

如果我们想导出使用缩进格式的json，我们可以使用marshallident函数，这允许您传递一个额外的string参数来指定您希望缩进的内容。两个空格对吧，不是一个标签？

1

func MarshalIndent(v interface{}, prefix, indent string) ([]byte, error)

精明的读者可能已经注意到，我们正在将结构解码为字节数组，然后将其写入响应流，这似乎不是特别有效，实际上也不是。go提供编码器和解码器，它们可以直接写入流，因为我们已经有了一个带有Responsewriter的流，那么让我们这样做吧。

如果我们有一个Responsewriter接口，我们如何将它与fmt.Fprint(w io.Writer, a ...interface{})一起使用？这个方法需要一个writer接口作为参数，我们有一个Responsewriter接口。如果我们看一下io.Writer，我们会发现它是：

1

Write(p []byte) (n int, err error)

因为Responsewriter接口实现了这个方法，所以它也满足接口编写器的要求，因此实现Responsewriter的任何对象都可以传递给任何需要Writer的函数。

太棒了，太棒了，但是我们还没有回答我们的问题，有没有更好的方法可以在返回数据之前将数据发送到输出流而不marshalling处理到临时对象？

encoding/json包有一个名为newencoder的函数，它返回一个Encoder对象，可用于将json直接写入打开的writer并猜测是什么；我们有一个：

// NewEncoder returns a new encoder that writes to w.
func NewEncoder(w io.Writer) *Encoder {
	return &Encoder{w: w, escapeHTML: true}
}

因此，我们可以直接将marshal的输出写入http响应，而不是将其存储到字节数组中。

解组json为go结构

现在我们已经了解了如何将json发送回客户机，如果在返回输出之前需要读取输入怎么办？我们可以使用url参数，我们将在下一章中看到这一切，但是通常您需要更复杂的数据结构，这些数据结构涉及到接受json作为http post请求的一部分的服务。

应用我们在上一节中学习到的类似技术来编写json，读取json同样简单。要将json解码为stuct字段，encoding/json包提供了Unmarshal函数：

1

func Unmarshal(data []byte, v interface{}) error

Unmarshal函数的工作方式与marshal相反；它根据需要分配maps，slices，和 pointers 。传入的object keys 使用结构字段名或其标记进行匹配，并将使用不区分大小写的匹配；但是，最好使用完全匹配。与marshal一样，解组将只设置导出的结构字段，这些字段以大写字母开头。

我们首先添加一个新的struct字段来表示请求，而Unmarshal可以将json解码为interface{}，对于JSON objects type 类型，接口是map[string]interface{}，对于JSON arrays，接口是：[]interface{}，这取决于我们的json是对象还是数组。

在我看来，如果我们以请求的形式显式地声明我们所期望的内容，代码的读者会更清楚。当我们来使用它，我们还可以节省自己的工作，不必手动铸造数据。

记住两件事：

  • 你不是为编译器编写代码，而是为人类编写代码

  • 读代码的时间比写代码的时间要长

考虑到这两点，我们创建了一个简单的结构来表示我们的请求，它将如下所示：
